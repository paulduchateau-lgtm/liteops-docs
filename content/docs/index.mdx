## Bienvenue

LiteOps construit des **systèmes intelligents d'opérations** composés d'opérateurs, d'agents et de systèmes d'orchestration.

Cette documentation couvre :

- **Opérateurs** (`OP-XXX`) — Briques fonctionnelles atomiques
- **Agents** (`AG00X`) — Compositions d'opérateurs résolvant un problème métier
- **Systèmes** (`SYS-XX`) — Orchestrations de plusieurs agents

## Architecture

```
Opérateurs (OP-XXX)
    │
    ├── OP-001 Data Layer
    ├── OP-002 LLM
    ├── OP-003 RAG
    ├── OP-004 OCR
    ├── OP-005 Doc Understanding
    ├── OP-006 Text to SQL
    ├── OP-007 Graph Gen
    ├── OP-008 Embedding
    ├── OP-009 NLP Parser
    ├── OP-010 Match Engine
    └── OP-011 Scorer

Agents (AG00X)
    │
    ├── AG001 Pilot      — Data Analytics par chatbot
    ├── AG002 Sailor     — RAG documentaire, zéro hallucination
    └── AG003 Matchmaker — Matching ressources–besoins

Systèmes (SYS-XX)
    │
    └── SYS-01 — Orchestration multi-agents
```

## Format d'échange : DataPacket

Tout opérateur communique via un **DataPacket** standardisé :

| Champ | Type | Description |
|-------|------|-------------|
| `id` | `string` | Identifiant unique |
| `version` | `string` | Version du packet |
| `quality_flag` | `PASS \| WARNING \| LOW` | Indicateur qualité |
| `quality_score` | `number` | Score 0–100 |
| `schema` | `object` | Schéma des données |
| `payload` | `unknown` | Données transportées |
| `lineage` | `array` | Traçabilité source + timestamp |

## Quality Flags

| Flag | Score | Signification |
|------|-------|---------------|
| `PASS` | 80+ | Données exploitables directement |
| `WARNING` | 50–79 | Données utilisables avec précaution |
| `LOW` | 0–49 | Données insuffisantes, action requise |
